计算机网络概述

概述

  1. 计算机网络的特点：

    1. 连通性

      无论距离多远，都好像直接连通在一起，能够交换信息

    2. 共享

      各种在互联网上的资源都能够被任何人直接读取或者下载使用

  2. 互联网+：互联网加各个传统行业

计算机网络

  1. 计算机网络包括节点和链路
    1. 节点：计算机、集线器、等等
    2. 链路：网线，光纤
  2. 互联网：网络的网络：将多个网络用路由器连接起来

  1. 主机host：与网络相连的计算机，并不狭隘的只包括计算机

  2. 历史

    1. 阿帕网ARPANET

      一开始用来实验网络，所有的计算机都就近地接在交换机节点上，完全的单一网络

    2. 互联网的出现

      internet：互连网，泛指计算机网络

      Internet：互联网，指我们现在用的Internet，采用TCP/IP

    3. 三级结构的互联网

      主干网，地区网，校园网

      并将经营权交给企业

    4. 多层次的ISP结构互联网

      ISP internet service provider：互联网服务提供商

      呈现一个树状的结构

      本地ISP→地区ISP→主干ISP

      但通过上级的服务一般都会需要收费，因此为了效率和经济，有的时候会在同级之间直接进行连接，也就是IXP(internet exchange point)。

      普通的用户上网就是借助ISP分配给他IP，然后接入自身，然后也就能访问网上的东西了，也就上网了。

      有的内容提供者需要大量的上行宽带，因此可能就会直接和高层ISP相连，或多个ISP相连，以加速。

  3. 互联网的标准

    1. 由互联网协会来管理→互联网体系结构委员会IAB

      1. 互联网工程部IETF→互联网工程指导小组IESG，短期或中期的标准化或问题
      2. 互联网研究部IRTF→互联网研究指导小组IRSG，长期的标准化或问题

    2. RFC(request for comments)

      所有的标准都会发布到网上，这些被成为RFC。

    3. 成为标准的过程

      1. 互联网草案

      2. 建议标准(成为RFC)

      3. 互联网标准(正式成为标准)

        被赋予STD的开头

互联网的组成

  1. 边缘部分：连接在互联网上的所有主机（端系统）。

    通过核型部分提供的服务互相通信。

    而他们的通信也实际上并不是计算机通信，而是其上的进程之间的通信。

    1. 客户-服务器

      客户：环境不需要很复杂，但需要知道服务器的地址。

      服务器：可同时处理多个客户，并且不需要知道服务器的地址。

      在建立连接后，通信是双向的。

    2. 对等连接P2P

      其实也可以看作是客户服务器模式，只不过都各自既做服务器也做客户端。

  2. 核心部分：路由器（一种专用计算机，转发收到的分组）

    1. 电路交换

      各个电话通过很多交换机连接到一起。每一次通话就是在端到端建立一条独占的物理通路。

      流程：建立连接→通话（其他人不能在此时使用占用的这条线路）→释放连接

      如果此时资源不足，就不会接受连接。

      而由于计算机的通信一般都不是连续的，因此这种方式的利用效率很低。

    2. 分组交换（存储转发）

      报文：要发送的完整数据

      分组：发送的时候将报文划分后增加首部控制信息后的包

      首部中包括了传输相关的所有信息

      主机：信息处理

      路由器：

      1. 转发分组

        需要知道该如何转发

      2. 分组交换

        将接收到的分组保存在内存中（相较于保存在磁盘更快），然后再转发出去。

      只在这条线路上传输的时候占用这条线路，做到了分时复用。

      而当出现故障或拥塞的时候，也可以自动切换到更好的路径（通过网状拓扑做到）。

      缺点：

      1. 转发需要排队（查找转发表和发送），产生时延。
      2. 额外的首部造成的开销，以及控制上的复杂。

    3. 三种传输方式比较

      当传输的数据远小于建立连接花的时间时，用报文或者分组交换，反之则电路交换更优。

      但是现代所有都基本使用分组交换了。

计算机网络在我国的发展

1980年开始联网实验。1989年第一个公用分组交换网CNPAC运行。1994年正式接入互联网。

各种网站也加快了发展：搜狐，网易，新浪，腾讯，百度，阿里巴巴。

计算机网络的类别

  1. 一个较好的对计算机网络的定义：由一些通用可编程的不是专门用来实现某一特定目的，且能传输多种不同类型数据的硬件

  2. 按作用范围分类

    1. 广域网WAN：是互联网的核心，一般是长距离高速率通信
    2. 城域网MAN：城市
    3. 局域网LAN：校园网和企业网都是局域网
    4. 个人区域网PAN：将个人工作的电子设备相连

    但如果距离非常近，如1米。则称为多处理机系统。

  3. 按网络的使用者进行分类

    1. 公用网：如电信公司建的网络，所有愿意缴纳费用的都可以使用网络。
    2. 专用网：为满足专用需求而建造的网络

  4. 用来把用户接入到互联网的网络

    接入网AN：就是本地ISP（将用户的端系统接入到互联网的ISP），不是边缘也不是核心，而是其中的交界处。

计算机网络的性能

  1. 速率：数据的传输速率（数据率，比特率），一般速率指的是额定速率，而非实际速率。

  2. 带宽：

    1. 原指：信号具有的频带宽度(Hz)，即允许哪些频率通过。
    2. 计算机网络：表示某通道传输数据的能力，表示单位时间类能通过的最高数据率。其实也就是速率。

    买的额定带宽只能保证从自己这到ISP的某个路由器的带宽，但不能保证后续在网络中的带宽。

  3. 吞吐量：单位时间内通过某个网络的实际数据量。

  4. 时延

    1. 发送时延：数据长度发送速率\frac{数据长度}{发送速率}发送速率数据长度​
    2. 传播时延：信道长度电磁波速度\frac{信道长度}{电磁波速度}电磁波速度信道长度​
    3. 处理时延：主机或路由器收到分组后需要的处理时间
    4. 排队时延：路由器中排队的时长，若溢出，丢失，则可认为时延无限大

    总时延：∑4个时延\sum4个时延∑4个时延

    时延和速率本身完全没有任何关系。

  5. 时延带宽积=传播时延✖️带宽，表示这段链路能容纳多少数据。称为以比特为单位的链路长度。

  6. 往返时间RTT： 2*传播时延+B发送+B排队+B处理，从发送完成开始记录，到接受完为止

  7. 信道利用率：有百分之几的时间信道是有数据通过的。

    然而并不是信道利用率越大越好。利用率越高，时延越大。D=D01−UD=\frac{D_0}{1-U}D=1−UD0​​，D0为空闲时的时延。当利用率超过0.5，时延就要加倍了，因此一般会控制在0.5以下。

  8. 其他：费用，质量，标准化，可靠性，可扩展性和可升级性，易于管理和维护

计算机网络体系结构

计算机网络要干的事情太多太杂，因此要分层。

  1. 历史

    最早是IBM宣布了系统网络体系结构SNA。

    接着是ISO提出开放系统互连基本参考模型OSI/RM，简称OSI。

    但基于TCP/IP的互联网率先成功，成为了事实上的标准。

  2. 协议

    规定了所交换的数据的格式以及有关的同步（时序）问题。

    包含3个要素：语法，语义，同步。

    通过划分层次，可以做到 各层之间是独立的，灵活性好，结构上可分割，易于实现和维护，能促进标准化工作。

    但他也会导致诸如相同的功能可能会重复出现的缺点。

  3. 体系结构

    1. 应用层：定义应用进程间的通信和交互的规则。HTTP，DNS，SMTP等。数据称为报文。
    2. 运输层：定义进程之间的信息传输规则，TCP，UDP。数据分别为报文段和用户数据报。
    3. 网络层（网际层/IP层）：定义主机之间的信息传输规则，IP。数据为IP数据包。
    4. 数据链路层：定义链路之间的信息传输规则。数据为帧。
    5. 物理层：传输的为比特。

    不管哪一层，数据都可用分组来表示。

    向下的每一层都会增加一个首部，往上每一层就会去掉一个首部。而路由器与最上面两层无关。

    对等层：相同高度的层级

    PDU：协议数据单元，对等层之间传输的数据单位。

  4. 其他

    实体：任何可发送或接受信息的硬件或软件进程

    协议：两个对等实体进行通信的规则的集合

    下面的协议对上面的实体是透明的。

    下层服务会向上提供服务（服务原语），上层会通过服务访问点（SAP）访问下面的服务。

    层与层之间交换的数据的单位称为服务数据单位SDU，服务数据单元。

    计算机网络的协议还有一个特点就是会把所有可能的情况都考虑到，而不是认为某件事发生的概率很小就不管了。

    核心部分，也就是网际层，非常简单，而复杂的功能放到边缘实现。

    PDU：本层传送的数据单元

    SDU：本层要交给上层的数据单元（不包括首部）

    协议控制信息（PCI）：本层的首部

    OSI中有好多重复的功能，并且限制得过于死了，TCP/IP就去掉了这些。
